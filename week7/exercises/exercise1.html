Exercise 1: Responsive Images

Task:

Create a new HTML file called "responsive-images.html" in the exercises folder. In this exercise, you'll practice implementing responsive image techniques to serve optimized images for different screen sizes and resolutions.

  1. Find or create three versions of the same image in different resolutions:
    • Small (400px width)
    • Medium (800px width)
    • Large (1200px width)
  2. Implement the <picture> element to serve different image sizes based on viewport width
  3. Implement the srcset and sizes attributes on a separate image
  4. Create an art direction example where the image crop changes for different screen sizes
  5. Implement lazy loading for at least one image
  6. Include proper alt text for all images

Hint:

Remember that the <picture> element is best for art direction (changing the image crop or completely different images based on screen size), while srcset and sizes attributes are better for resolution switching (same image, different resolutions).

Expected Result

Your page should demonstrate three different responsive image techniques:

  1. Resolution switching with srcset and sizes
  2. Art direction with the <picture> element
  3. Performance optimization with lazy loading

                    (max-width: 1200px) 800px,
                    1200px">
    </section>
    
    <section>
        <h2>2. Art Direction with picture Element</h2>

        </picture>
    </section>

        <h2>3. Lazy Loading</h2>

</html>

Check Your Work

After completing the exercise:

  1. Open your HTML file in a web browser
  2. Resize the browser window to test your responsive images
  3. Use the browser's developer tools to verify different images are being loaded at different screen sizes
  4. Check that lazy loading is working by scrolling down to the lazy-loaded image
  5. Validate your HTML using the W3C Markup Validation Service

Submission Guidelines

When submitting your work, be prepared to:

  1. Explain how each responsive image technique works
  2. Discuss the benefits and use cases for each approach
  3. Demonstrate how your implementation performs on different screen sizes
