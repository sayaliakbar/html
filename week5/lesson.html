Week 5: HTML5 Semantic Elements

Introduction to Semantic Elements

HTML5 introduced several semantic elements that provide meaning to the structure of a web page. These elements help improve accessibility, SEO, and code readability.

Semantic elements clearly describe their meaning in a human- and machine-readable way. Examples include <header>, <footer>, <article>, and <section>.

Container Semantics

Header, Footer, Nav, and Main Elements

The <header> element represents introductory content, typically a group of introductory or navigational aids. 

<header>
    <h1>Website Title</h1>
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>
</header>

The <footer> element represents a footer for its nearest sectioning content or sectioning root element. 

<footer>
    <p>© 2025 My Website. All rights reserved.</p>
    <p>Contact: <a href="mailto:info@example.com">info@example.com</a></p>
</footer>

The <nav> element represents a section of a page that links to other pages or to parts within the page. 

<nav>
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

The <main> element represents the dominant content of the <body> of a document. There should be only one <main> element per document. 

<main>
    <h2>Main Content</h2>
    <p>This is the main content area.</p>
</main>

Content Semantics

Article, Section, and Aside Elements

The <article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable. 

<article>
    <h2>Blog Post Title</h2>
    <p>Published on <time datetime="2025-01-15">January 15, 2025</time></p>
    <p>Article content...</p>
</article>

The <section> element represents a standalone section of content that could have its own heading. 

<section>
    <h2>Features</h2>
    <p>Content about features...</p>
</section>

The <aside> element represents a portion of a document whose content is only indirectly related to the document's main content. 

<aside>
    <h3>Related Information</h3>
    <p>Sidebar content...</p>
</aside>

Specialized Semantic Elements

Figure, Time, and Address Elements

The <figure> element represents self-contained content, potentially with an optional caption, which is specified using the <figcaption> element. 

<figure>
    <img src="image.jpg" alt="Description of the image">
    <figcaption>Caption for the image</figcaption>
</figure>

The <time> element represents a specific period in time. 

<time datetime="2025-01-15">January 15, 2025</time>

The <address> element supplies contact information for its nearest <article> or <body> element ancestor. 

<address>
    <p>Contact us at <a href="mailto:info@example.com">info@example.com</a></p>
</address>
