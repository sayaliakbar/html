Week 4: HTML Structure

Block vs. Inline Elements

HTML elements are generally classified as either block-level elements or inline elements. Understanding the difference is crucial for creating well-structured web pages.

Block-Level Elements

Block-level elements have these characteristics:

  • Start on a new line
  • Take up the full width available
  • Create a "block" in the flow of the document
  • Can contain other block elements and inline elements

Common Block-Level Elements:

  • <div> - Generic container for flow content
  • <h1> through <h6> - Headings
  • <p> - Paragraph
  • <ul>, <ol> - Lists
  • <li> - List item
  • <table> - Table
  • <form> - Form
  • <header>, <footer>, <section>, <article> - Semantic containers

Inline Elements

Inline elements have these characteristics:

  • Do not start on a new line
  • Take up only as much width as necessary
  • Flow within the text content
  • Cannot contain block-level elements

Common Inline Elements:

  • <span> - Generic inline container
  • <a> - Anchor/link
  • <strong>, <em> - Text emphasis
  • <img> - Image
  • <br> - Line break
  • <input>, <button> - Form controls
  • <code> - Code snippet
  • <i>, <b> - Italics, bold

Changing Display Behavior

It's important to understand that the default display behavior of HTML elements can be changed using CSS: 

/* CSS to change display behavior */
span {
    display: block;
}
div {
    display: inline;
}

Understanding the difference between block and inline elements is crucial for layout, especially when using CSS for styling.

Divs and Spans

The <div> and <span> elements are generic containers that help organize and group content for styling and structural purposes.

The Div Element

The <div> (division) element is a block-level container that is used to group larger sections of content: 

<div>
    <h2>Section Title</h2>
    <p>This is a paragraph inside a div.</p>
    <p>Another paragraph in the same section.</p>
</div>

Uses for Divs:

  • Creating logical sections or containers
  • Grouping elements for styling with CSS
  • Building layout structures
  • Creating reusable components

Before HTML5, divs were the primary way to structure web pages. With HTML5, we have more semantic elements like <header>, <nav>, etc., but divs are still useful for generic containers.

The Span Element

The <span> element is an inline container used to group small pieces of content within a line: 

<p>The temperature today is <span class="highlight">75°F</span> and sunny.</p>

Uses for Spans:

  • Applying styles to a portion of text
  • Adding specific attributes to text segments
  • Targeting small pieces of content with JavaScript
  • Creating inline-level hooks for CSS/JS

Difference Between Divs and Spans

Feature Div Span
Display type Block-level Inline
Line break Creates new lines No line breaks
Width Takes full width available Takes only needed width
Typical use Larger sections of content Small portions of text
Can contain block elements Yes No

Always choose the most semantically appropriate HTML element for your content. Use <div> and <span> when no other semantic element is available.

Basic Page Layouts

Creating effective page layouts is an essential part of web development. While CSS is responsible for the visual layout, HTML provides the structure that CSS works with.

Common Layout Patterns

Several layout patterns are commonly used in web design:

1. Header, Content, Footer

<div class="page-container">
    <div class="header">
        <h1>Website Title</h1>
    </div>
    
    <div class="content">
        <p>Main content goes here...</p>
    </div>
    
    <div class="footer">
        <p>© 2025 My Website</p>
    </div>
</div>

2. Two-Column Layout

<div class="container">
    <div class="header">
        <h1>My Website</h1>
    </div>
    
    <div class="main-content">
        <div class="sidebar">
            <h2>Navigation</h2>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
                <li><a href="#">Contact</a></li>
            </ul>
        </div>
        
        <div class="content">
            <h2>Main Content</h2>
            <p>This is the main content area.</p>
        </div>
    </div>
    
    <div class="footer">
        <p>© 2025 My Website</p>
    </div>
</div>

Layout Considerations

  • Content organization: Group related content together
  • Hierarchy: Use headings to establish a clear content hierarchy
  • Navigation: Ensure your layout accommodates easy navigation
  • Responsive design: Consider how the layout will adapt to different screen sizes
  • Accessibility: Make sure your layout is accessible to all users

HTML5 Structural Elements

HTML5 introduced several semantic elements that help create more meaningful page structures. These elements better describe the purpose of the content they contain.

Key HTML5 Structural Elements

<header>

Represents introductory content, typically a group of introductory or navigational aids. 

<header>
    <h1>Website Name</h1>
    <p>Slogan or tagline</p>
</header>

<nav>

Represents a section of a page that contains navigation links.

<nav>
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

<main>

Represents the main content area of the document. There should be only one <main> element per page. 

<main>
    <h2>Article Title</h2>
    <p>Main content goes here...</p>
</main>

<article>

Represents a self-contained composition that could be distributed independently. 

<article>
    <h2>Blog Post Title</h2>
    <p>Published on <time datetime="2025-01-15">January 15, 2025</time></p>
    <p>Article content...</p>
</article>

<section>

Represents a standalone section of content that could have its own heading. 

<section>
    <h2>Features</h2>
    <p>Content about features...</p>
</section>

<aside>

Represents content that is tangentially related to the content around it. 

<aside>
    <h3>Related Information</h3>
    <p>Sidebar content...</p>
</aside>

<footer>

Represents a footer for a document or section.

<footer>
    <p>© 2025 My Website. All rights reserved.</p>
    <p>Contact: <a href="mailto:info@example.com">info@example.com</a></p>
</footer>

Comparing Div-based vs. Semantic Structure

Old Approach (Div-based):

<div class="header">
    <h1>Website Title</h1>
</div>
<div class="nav">
    <!-- Navigation -->
</div>
<div class="content">
    <div class="article">
        <!-- Article content -->
    </div>
    <div class="sidebar">
        <!-- Sidebar content -->
    </div>
</div>
<div class="footer">
    <!-- Footer content -->
</div>

New Approach (Semantic):

<header>
    <h1>Website Title</h1>
</header>
<nav>
    <!-- Navigation -->
</nav>
<main>
    <article>
        <!-- Article content -->
    </article>
    <aside>
        <!-- Sidebar content -->
    </aside>
</main>
<footer>
    <!-- Footer content -->
</footer>

Benefits of Semantic Structure

  • Accessibility: Screen readers understand the page structure
  • SEO: Search engines better understand your content
  • Maintainability: Code is more readable and easier to maintain
  • Consistency: Encourages consistent structure across pages
  • Future-proof: Better prepared for future web technologies

While HTML5 semantic elements provide meaning to the structure, they don't automatically create visual layouts. You'll still need CSS to define how these elements are displayed.
