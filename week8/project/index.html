Week 8 Project: Metadata-Enhanced Website

Apply what you've learned about metadata and API integration

Project Overview

In this project, you'll create a small website (3-5 pages) for a fictional business or organization that implements comprehensive metadata and third-party API integrations. The focus is on optimizing your website for search engines, social sharing, and enhanced functionality through embedded content.

This project will synthesize the concepts learned throughout Week 8, giving you practical experience implementing metadata strategies and integrating external services.

Project Requirements

Basic Requirements:

  1. Create a multi-page website (minimum 3 pages) with consistent navigation
  2. Implement proper HTML document structure and semantic elements
  3. Include a responsive design that works on different screen sizes
  4. Add appropriate metadata to improve SEO and social sharing
  5. Integrate at least two different third-party APIs or embedded widgets

Metadata Requirements:

  1. Each page must include:
    • Unique and descriptive title (under 60 characters)
    • Meta description (under 160 characters)
    • Open Graph metadata for Facebook and other platforms
    • Twitter Card metadata
    • Favicon information
  2. At least one page must include Schema.org structured data (using either Microdata or JSON-LD) appropriate to the page content (e.g., local business, product, article, event)
  3. Include canonical URLs for all pages
  4. Implement appropriate robots meta tags

API Integration Requirements:

  1. Implement at least two of the following:
    • Google Maps or other mapping service
    • Social media feed or sharing buttons
    • Weather widget
    • YouTube or other video embed
    • Calendar or event integration
    • Contact form
    • Other appropriate API integration
  2. All embedded content must:
    • Include appropriate error handling or fallbacks
    • Be responsive and work on different screen sizes
    • Load efficiently (e.g., using lazy loading where appropriate)
    • Include proper attribution if required
  3. Add a privacy notice about third-party content

Project Steps

  1. Planning:
    • Choose a business or organization type (restaurant, retail store, nonprofit, etc.)
    • Determine the purpose and content for each page
    • Select which APIs or embedded content you'll integrate
    • Plan your metadata strategy for each page
  2. Building the Basic Structure:
    • Create the HTML files for each page with proper document structure
    • Implement navigation between pages
    • Add basic CSS styling for layout and responsiveness
  3. Adding Content:
    • Add text content, placeholder images, and other basic page elements
    • Structure content using semantic HTML elements
  4. Implementing Metadata:
    • Add basic metadata (title, description, charset, viewport) to all pages
    • Implement Open Graph and Twitter Card metadata
    • Add structured data to appropriate pages
  5. Integrating APIs:
    • Add your chosen API integrations to the appropriate pages
    • Implement error handling and fallback content
    • Ensure embedded content is responsive
  6. Testing and Refinement:
    • Validate your HTML using the W3C Validator
    • Test your metadata using tools like the Facebook Sharing Debugger and Twitter Card Validator
    • Verify your structured data using Google's Rich Results Test or Schema.org Validator
    • Test your website on different devices and screen sizes
    • Ensure all embedded content has proper fallbacks

Project Resources

  • Facebook Sharing Debugger
  • Twitter Card Validator
  • Google Rich Results Test
  • Schema.org Validator

API Documentation:

  • Google Maps Embed API
  • OpenWeather API
  • Facebook Social Plugins
  • Twitter for Websites
  • YouTube IFrame Player API

Project Tips

  • For metadata: Create a template for your metadata to ensure consistency across pages, then customize the specific values for each page.
  • For images: Prepare properly sized images for Open Graph and Twitter Cards in advance.
  • For APIs: Start with sample code from the API documentation and modify it to fit your needs.
  • For testing: Test frequently during development, especially after adding new APIs or metadata.
  • For structured data: Use a tool like the Google Structured Data Markup Helper to get started, then refine your implementation.
  • For error handling: Always implement fallbacks for API content in case the external service is unavailable.

Project Example

Here's an example of a meta section for a page about a local bakery: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Artisan Breads & Pastries | Sweet Morning Bakery</title>
    <meta name="description" content="Discover our handcrafted artisan breads and freshly baked pastries made with locally sourced organic ingredients at Sweet Morning Bakery.">
    <link rel="canonical" href="https://sweetmorningbakery.example.com/products.html">
    
    <!-- Open Graph Metadata -->
    <meta property="og:title" content="Artisan Breads & Pastries | Sweet Morning Bakery" />
    <meta property="og:description" content="Handcrafted artisan breads and pastries made with locally sourced organic ingredients. Visit our bakery today!" />
    <meta property="og:image" content="https://sweetmorningbakery.example.com/images/bakery-products-social.jpg" />
    <meta property="og:url" content="https://sweetmorningbakery.example.com/products.html" />
    <meta property="og:type" content="website" />
    <meta property="og:site_name" content="Sweet Morning Bakery" />
    
    <!-- Twitter Card Metadata -->
    <meta name="twitter:card" content="summary_large_image" />
    <meta name="twitter:site" content="@sweetmorningbakery" />
    <meta name="twitter:title" content="Artisan Breads & Pastries | Sweet Morning Bakery" />
    <meta name="twitter:description" content="Handcrafted artisan breads and pastries made with locally sourced organic ingredients. Visit our bakery today!" />
    <meta name="twitter:image" content="https://sweetmorningbakery.example.com/images/bakery-products-social.jpg" />
    
    <!-- Favicon -->
    <link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
    <link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
    <link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
    <link rel="manifest" href="/site.webmanifest">
    
    <!-- Robots -->
    <meta name="robots" content="index, follow">
</head>
