Exercise 2: Improving Page Structure with Semantic HTML

Introduction

In this exercise, you will practice improving web page structures using semantic HTML elements. Semantic HTML improves accessibility by providing more context to assistive technologies like screen readers.

For each example, you will be given non-semantic HTML and asked to rewrite it using proper semantic elements.

Convert this basic webpage structure from non-semantic to semantic HTML:

Task 4: Form Structure

Convert this contact form from non-semantic to semantic HTML:

Before (Non-Semantic HTML)

<div class="contact-form">
  <div class="form-title">Contact Us</div>
  <div class="form-description">
    Fill out this form to get in touch with our team.
  </div>
  
  <div class="form-container">
    <div class="form-row">
      <div class="form-label">Name:</div>
      <div class="form-input">
        <input type="text" name="name">
      </div>
    </div>
    
    <div class="form-row">
      <div class="form-label">Email:</div>
      <div class="form-input">
        <input type="text" name="email">
      </div>
    </div>
    
    <div class="form-row">
      <div class="form-label">Subject:</div>
      <div class="form-input">
        <select name="subject">
          <option value="">Choose a subject</option>
          <option value="support">Customer Support</option>
          <option value="sales">Sales Inquiry</option>
          <option value="other">Other</option>
        </select>
      </div>
    </div>
    
    <div class="form-row">
      <div class="form-label">Message:</div>
      <div class="form-input">
        <textarea name="message" rows="5"></textarea>
      </div>
    </div>
    
    <div class="form-row">
      <div class="form-input">
        <input type="checkbox" name="subscribe">
        <span>Subscribe to newsletter</span>
      </div>
    </div>
    
    <div class="form-actions">
      <div class="submit-button">Send Message</div>
      <div class="reset-button">Clear Form</div>
    </div>
  </div>
</div>

After (Your Semantic HTML)

Tip: Forms have several semantic elements that improve accessibility: <form>, <fieldset>, <legend>, <label> (with proper for attributes), and appropriate input types. Also consider adding ARIA attributes where needed.

General Tips for Semantic HTML:

  • Always choose the most appropriate semantic element for the content's meaning, not its appearance
  • Use a logical heading structure (h1-h6) that correctly represents the document outline
  • Ensure all interactive elements are using the correct elements (buttons should be <button>, not <div>)
  • Use lists (<ul>, <ol>) for collections of related items
  • Add appropriate landmark elements to help screen reader users navigate the page
  • Include descriptive alt text for images
  • Use <figure> and <figcaption> for images with captions
  • Remember that using semantic HTML is the foundation of web accessibility
