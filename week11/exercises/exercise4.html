Exercise 4: Creating Keyboard-Accessible Components

Introduction

In this exercise, you will practice making interactive components accessible to keyboard-only users. Many people with motor disabilities, as well as screen reader users, rely on keyboards for navigation. Ensuring keyboard accessibility is crucial for creating inclusive web experiences.

For each component, you will:

  1. Implement keyboard navigation
  2. Add appropriate focus management
  3. Ensure visible focus indicators
  4. Handle key events to match expected behavior

Keyboard Accessibility Tips:

  • All interactive elements should be focusable and operable with a keyboard
  • Focus order should be logical and follow the visual layout of the page
  • Focus indicators should be clearly visible (don't remove outlines without alternatives)
  • Follow standard keyboard interaction patterns: Enter/Space for activation, arrow keys for navigation, Escape for cancellation
  • Use tabindex="0" to make non-interactive elements focusable, and tabindex="-1" for programmatic focus
  • Avoid using positive tabindex values as they disrupt the natural tab order
  • Test your components using only keyboard navigation

After completing these exercises, you'll have gained valuable experience in creating keyboard-accessible components, which is a crucial aspect of web accessibility.
