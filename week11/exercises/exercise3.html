Exercise 3: Implementing ARIA for Complex Widgets

Introduction

In this exercise, you will practice applying WAI-ARIA attributes to make complex interactive widgets more accessible. ARIA helps to communicate the purpose, state, and functionality of custom UI components to assistive technologies.

For each widget, you will:

  1. Examine the existing HTML structure
  2. Add appropriate ARIA roles, states, and properties
  3. Implement JavaScript to update ARIA states when the widget's state changes

General ARIA Tips:

  • Always follow the first rule of ARIA: if you can use a native HTML element with the semantics you need, prefer that over ARIA
  • ARIA doesn't change functionality; you still need JavaScript to implement behavior
  • Only use ARIA when necessary—don't add it to elements that already have appropriate semantics
  • Test your components with screen readers to ensure they're announcing information correctly
  • When a component's state changes, be sure to update relevant ARIA attributes
  • Be careful with keyboard navigation—ensure all interactive elements are focusable and respond to appropriate key events
