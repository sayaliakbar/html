Week 3: Forms and Input Basics

Form Structure

HTML forms are used to collect user input. The data can be sent to a server for processing or handled by JavaScript on the client side.

Basic Form Syntax

Forms are created using the <form> element:

<form action="/submit-form" method="post">
  <!-- Form elements go here -->
  <input type="text" name="username">
  <input type="submit" value="Submit">
</form>

Form Attributes

Attribute Description Example
action Specifies where to send the form data when submitted action="/process-form.php"
method Specifies the HTTP method to use when sending form data method="post" or method="get"
name Specifies a name for the form (for DOM manipulation) name="registration-form"
autocomplete Specifies whether form should have autocomplete on or off autocomplete="on" or autocomplete="off"
target Specifies where to display the response after submitting target="_blank" (new tab) or target="_self" (default)

GET vs POST Methods

  • GET:
    • Appends form data to the URL as query parameters
    • Limited to about 2000 characters
    • Not secure (data is visible in the URL)
    • Can be bookmarked
    • Best for non-sensitive data and search forms
  • POST:
    • Sends form data in the HTTP request body
    • No size limitations
    • More secure (data not visible in the URL)
    • Cannot be bookmarked
    • Best for sensitive data and large amounts of data

When a form is submitted, each form control with a name attribute will have its value sent to the server.

Input Types

The <input> element is the most commonly used form element. Its behavior changes significantly depending on the type attribute.

Basic Input Types

Text Input

<input type="text" name="username" placeholder="Enter your username">

Password Input

<input type="password" name="password" placeholder="Enter your password">

Radio Buttons

Radio buttons allow users to select one option from a group. All radio buttons in the same group must have the same name attribute. 

<input type="radio" name="gender" id="male" value="male">
<label for="male">Male</label>

<input type="radio" name="gender" id="female" value="female">
<label for="female">Female</label>

<input type="radio" name="gender" id="other" value="other">
<label for="other">Other</label>


Checkboxes

Checkboxes allow users to select multiple options.

<input type="checkbox" name="hobby" id="reading" value="reading">
<label for="reading">Reading</label>

<input type="checkbox" name="hobby" id="sports" value="sports">
<label for="sports">Sports</label>

<input type="checkbox" name="hobby" id="music" value="music">
<label for="music">Music</label>


Email Input

<input type="email" name="email" placeholder="Enter your email">

Number Input

<input type="number" name="age" min="18" max="100" placeholder="Enter your age">

Input Attributes

Attribute Description Example
name Specifies the name of the input (sent with the form data) name="username"
value Specifies the default value of the input value="John"
placeholder Provides a hint about what to enter in the input placeholder="Enter your username"
required Indicates that the input must be filled out required
disabled Makes the input non-interactive and not submitted disabled
readonly Makes the input non-editable but still submitted readonly
min/max Sets minimum and maximum values for number inputs min="0" max="100"

Text Area

For multi-line text input, use the <textarea> element: 

<textarea name="message" rows="5" cols="30" placeholder="Enter your message"></textarea>

Select Dropdown

The <select> element creates a dropdown list:

<select name="country">
  <option value="">Select a country</option>
  <option value="usa">United States</option>
  <option value="canada">Canada</option>
  <option value="uk">United Kingdom</option>
</select>

Buttons and Submit Controls

Buttons are interactive elements that allow users to submit forms or trigger actions.

Submit Button

The submit button sends the form data to the server:

<input type="submit" value="Submit Form">

Alternatively, you can use the <button> element with type="submit": 

<button type="submit">Submit Form</button>

Reset Button

The reset button clears all form inputs to their default values:

<input type="reset" value="Reset Form">

Or with the button element:

<button type="reset">Reset Form</button>

Button Element

The <button> element is more flexible than <input type="button"> because it can contain HTML content: 

<button type="button" onclick="alert('Button clicked!')">
  <strong>Click Me</strong>
  <span>I am a custom button</span>
</button>

Button Types

  • type="submit" - Submits the form
  • type="reset" - Resets the form
  • type="button" - No default behavior (used with JavaScript)

If you don't specify a type for a <button> element, it defaults to type="submit" when inside a form.

Image Button

You can also use an image as a submit button:

<input type="image" src="submit-button.png" alt="Submit" width="48" height="48">

Labels and Fieldsets

Labels and fieldsets improve form organization and accessibility.

Labels

The <label> element associates text with a specific form control: 

<!-- Method 1: Using the 'for' attribute -->
<label for="username">Username:</label>
<input type="text" id="username" name="username">

<!-- Method 2: Wrapping the input -->
<label>
  Password:
  <input type="password" name="password">
</label>


Benefits of Labels

  • Clicking the label focuses or activates the associated input
  • Improves accessibility for screen readers
  • Makes radio buttons and checkboxes easier to click
  • Provides a semantic connection between the label and input

Fieldsets and Legends

The <fieldset> element groups related form controls, and the <legend> element provides a caption for the fieldset: 

<fieldset>
  <legend>Contact Information</legend>
  
  <label for="fname">First name:</label>
  <input type="text" id="fname" name="fname"><br><br>
  
  <label for="lname">Last name:</label>
  <input type="text" id="lname" name="lname"><br><br>
  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email">
</fieldset>
Contact Information



Benefits of Fieldsets

  • Groups related form controls visually and semantically
  • Improves screen reader navigation
  • Makes complex forms more organized and user-friendly
  • Can be styled as a group

Fieldsets are especially useful for grouping related radio buttons and checkboxes, as well as dividing long forms into logical sections.

Form Accessibility

Creating accessible forms ensures that all users, including those with disabilities, can interact with your forms.

Accessibility Best Practices

  • Associate labels with form controls using the for attribute
  • Use descriptive labels that clearly explain the purpose of the input
  • Group related form elements with fieldsets and legends
  • Provide clear instructions and error messages
  • Use the required attribute for mandatory fields
  • Ensure keyboard navigation works correctly
  • Maintain a logical tab order
  • Provide feedback when a form is submitted successfully

ARIA Attributes

ARIA (Accessible Rich Internet Applications) attributes can enhance form accessibility: 

<label for="username">Username:</label>
<input type="text" id="username" name="username" aria-required="true" aria-describedby="username-help">
<p id="username-help">Username must be at least 5 characters.</p>

When designing forms, always consider accessibility from the beginning, not as an afterthought.
