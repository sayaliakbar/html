Week 2: Core HTML Elements

Links and Anchors

Hyperlinks are one of the most important elements in HTML. They allow users to navigate between web pages and resources.

Basic Link Syntax

Links are created using the <a> (anchor) element with the href attribute: 

<a href="https://www.example.com">Visit Example Website</a>

Types of Links

Absolute URLs

Absolute URLs point to resources using their complete web address:

<a href="https://www.example.com/about.html">About Page</a>

Relative URLs

Relative URLs point to resources relative to the current page:

<a href="about.html">About Page</a>
<a href="../index.html">Home Page</a>
<a href="./products/item1.html">Product 1</a>

Email Links

Email links open the user's email client with a pre-addressed message: 

<a href="mailto:contact@example.com">Email Us</a>

Phone Links

Phone links can initiate calls on mobile devices:

<a href="tel:+1234567890">Call Us</a>

Page Sections (Bookmarks)

You can link to specific sections within a page using IDs:

<!-- Creating a bookmark target -->
<h2 id="section1">Section 1</h2>

<!-- Linking to that bookmark -->
<a href="#section1">Go to Section 1</a>

<!-- Linking to a bookmark on another page -->
<a href="page.html#section1">Go to Section 1 on Page</a>

Link Attributes

Attribute Description Example
target Specifies where to open the linked document <a href="https://example.com" target="_blank">Open in new tab</a>
title Provides additional information about the link <a href="about.html" title="Learn more about our company">About Us</a>
download Indicates that the target will be downloaded <a href="report.pdf" download>Download Report</a>
rel Specifies the relationship between the current document and the linked document <a href="https://example.com" rel="noopener noreferrer">External Link</a>

For security and performance reasons, use rel="noopener noreferrer" when linking to external sites with target="_blank".

Images

Images make your web pages more engaging and informative. HTML uses the <img> element to embed images.

Basic Image Syntax

<img src="image.jpg" alt="Description of the image">

The src attribute specifies the path to the image, and the alt attribute provides alternative text for accessibility.

Image Formats

  • JPEG (.jpg, .jpeg) - Best for photographs and complex images with many colors
  • PNG (.png) - Best for images requiring transparency or with text/sharp edges
  • GIF (.gif) - Supports animation, limited colors
  • SVG (.svg) - Vector format, scales without losing quality, ideal for logos and icons
  • WebP (.webp) - Modern format with better compression and quality

Image Attributes

Attribute Description Example
alt Alternative text for the image (required for accessibility) <img src="logo.png" alt="Company Logo">
width / height Specifies the width/height of the image in pixels <img src="photo.jpg" alt="Beach" width="300" height="200">
loading Specifies whether the image should be loaded immediately or when needed <img src="large-image.jpg" alt="Landscape" loading="lazy">

Always specify the alt attribute for images. For decorative images, use an empty alt attribute (alt="").

Image Maps

Image maps allow you to create clickable areas within an image:

<img src="workplace.jpg" alt="Workplace" usemap="#workmap">

<map name="workmap">
  <area shape="rect" coords="34,44,270,350" alt="Computer" href="computer.html">
  <area shape="rect" coords="290,172,333,250" alt="Phone" href="phone.html">
  <area shape="circle" coords="337,300,44" alt="Coffee" href="coffee.html">
</map>

Area Shape Attributes

  • rect - Defines a rectangular area (coords: x1,y1,x2,y2)
  • circle - Defines a circular area (coords: x,y,radius)
  • poly - Defines a polygon (coords: x1,y1,x2,y2,...)

Lists

HTML provides three types of lists to organize content: unordered lists, ordered lists, and definition lists.

Unordered Lists

Unordered lists create bulleted lists using the <ul> element with <li> items: 

<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
</ul>

Example:

  • Item 1
  • Item 2
  • Item 3

Ordered Lists

Ordered lists create numbered lists using the <ol> element with <li> items: 

<ol>
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
</ol>

Example:

  1. First step
  2. Second step
  3. Third step

Ordered List Attributes

<!-- Starting number -->
<ol start="5">
  <li>Item (will be numbered 5)</li>
  <li>Item (will be numbered 6)</li>
</ol>

<!-- Different numbering types -->
<ol type="i">  <!-- lowercase Roman numerals -->
  <li>Item (i)</li>
  <li>Item (ii)</li>
</ol>

<ol type="A">  <!-- uppercase letters -->
  <li>Item (A)</li>
  <li>Item (B)</li>
</ol>

List Type Values

  • 1 - Numbers (default)
  • A - Uppercase letters
  • a - Lowercase letters
  • I - Uppercase Roman numerals
  • i - Lowercase Roman numerals

Definition Lists

Definition lists are used to display name-value pairs, such as terms and their definitions: 

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language is the standard language for creating web pages.</dd>
  
  <dt>CSS</dt>
  <dd>Cascading Style Sheets is used for styling web pages.</dd>
</dl>

Example:

HTML
HyperText Markup Language is the standard language for creating web pages.
CSS
Cascading Style Sheets is used for styling web pages.

Nested Lists

Lists can be nested inside other lists:

<ul>
  <li>Main Item 1</li>
  <li>Main Item 2
    <ul>
      <li>Sub Item 2.1</li>
      <li>Sub Item 2.2</li>
    </ul>
  </li>
  <li>Main Item 3</li>
</ul>

Example:

  • Main Item 1
  • Main Item 2
    • Sub Item 2.1
    • Sub Item 2.2
  • Main Item 3

Tables

HTML tables allow you to arrange data in rows and columns.

Basic Table Structure

<table>
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
  </tr>
  <tr>
    <td>Row 1, Cell 1</td>
    <td>Row 1, Cell 2</td>
  </tr>
  <tr>
    <td>Row 2, Cell 1</td>
    <td>Row 2, Cell 2</td>
  </tr>
</table>

Basic Table Elements

  • <table> - Defines the table
  • <tr> - Defines a table row
  • <th> - Defines a header cell
  • <td> - Defines a standard cell

Semantic Table Elements

<table>
  <caption>Monthly Savings</caption>
  <thead>
    <tr>
      <th>Month</th>
      <th>Savings</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>January</td>
      <td>$100</td>
    </tr>
    <tr>
      <td>February</td>
      <td>$80</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>Total</th>
      <td>$180</td>
    </tr>
  </tfoot>
</table>

Semantic Table Elements

  • <caption> - Defines a table caption or title
  • <thead> - Groups header content
  • <tbody> - Groups body content
  • <tfoot> - Groups footer content

Table Cell Spanning

<!-- Spanning columns -->
<tr>
  <td colspan="2">This cell spans two columns</td>
</tr>

<!-- Spanning rows -->
<tr>
  <td rowspan="2">This cell spans two rows</td>
  <td>Cell 2</td>
</tr>

Example of Column Spanning:

Name Age Country
John 25 USA
This cell spans all three columns
Maria 28 Canada

Accessibility Considerations

For better accessibility, add the scope attribute to header cells: 

<table>
  <tr>
    <th scope="col">Name</th>
    <th scope="col">Age</th>
  </tr>
  <tr>
    <th scope="row">John</th>
    <td>25</td>
  </tr>
</table>

Tables should only be used for tabular data, not for page layout. Use CSS for layout purposes.
