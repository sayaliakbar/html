Week 2: Core HTML Elements

1. Links and Anchors

Links are fundamental to the web, allowing users to navigate between pages and resources.

Basic Link Syntax

Links are created using the <a> (anchor) element with the href attribute:

<a href="https://www.example.com">Visit Example.com</a>

Types of Links

  • Absolute URLs: Point to external websites, include the full URL
    <a href="https://www.mozilla.org">Mozilla</a>
  • Relative URLs: Point to pages within your website
    <a href="about.html">About Us</a>
    <a href="../products/index.html">Products</a>
  • Email Links: Open the user's email client
    <a href="mailto:example@example.com">Send Email</a>
  • Phone Links: For mobile devices to initiate calls
    <a href="tel:+1234567890">Call Us</a>
  • Page Anchors: Link to a specific part of a page
    <a href="#section2">Go to Section 2</a>
    <h2 id="section2">Section 2</h2>

Link Attributes

Attribute Description Example
target Specifies where to open the linked document <a href="https://example.com" target="_blank">Open in new tab</a>
download Indicates the link is for downloading a file <a href="report.pdf" download>Download Report</a>
rel Describes the relationship between current and linked document <a href="https://partner.com" rel="nofollow">Partner Site</a>
title Provides additional information (tooltip) <a href="glossary.html" title="View our glossary">Glossary</a>

2. Images and Image Maps

Images enhance the visual appeal of web pages and convey information effectively.

Basic Image Syntax

Images are added using the <img> element with the src attribute:

<img src="image.jpg" alt="Description of image">

Important Image Attributes

Attribute Description Example
alt Alternative text for accessibility and if image fails to load <img src="logo.png" alt="Company Logo">
width, height Specify image dimensions (good for performance) <img src="photo.jpg" width="300" height="200" alt="Landscape">
loading Lazy loading for performance optimization <img src="large-image.jpg" alt="Large image" loading="lazy">

Image Formats

  • JPEG: Best for photographs with many colors
  • PNG: Good for images needing transparency
  • GIF: Simple animations and images with few colors
  • SVG: Vector graphics that scale without losing quality
  • WebP: Modern format with better compression

Image Maps

Image maps allow different parts of an image to link to different destinations:

<img src="map.jpg" alt="Navigation Map" usemap="#workmap"> <map name="workmap"> <area shape="rect" coords="34,44,270,350" alt="Computer" href="computer.html"> <area shape="rect" coords="290,172,333,250" alt="Phone" href="phone.html"> <area shape="circle" coords="337,300,44" alt="Cup of coffee" href="coffee.html"> </map>

3. Lists

Lists help organize content in a structured way, making information easier to read.

Unordered Lists

Use bullet points for items with no particular order:

<ul> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ul>

Ordered Lists

Use numbers for items that follow a sequence:

<ol> <li>First step</li> <li>Second step</li> <li>Third step</li> </ol>

Definition Lists

Used for term-description pairs:

<dl> <dt>HTML</dt> <dd>HyperText Markup Language - the standard language for web pages</dd> <dt>CSS</dt> <dd>Cascading Style Sheets - used for styling web pages</dd> </dl>

Nested Lists

Lists can be nested inside each other:

<ul> <li>Main item 1 <ul> <li>Sub item 1.1</li> <li>Sub item 1.2</li> </ul> </li> <li>Main item 2</li> </ul>

List Attributes

Element Attribute Description
ol type Number type: 1, A, a, I, i
ol start Starting number for ordered list
ol reversed List in descending order
li value Set specific number for list item

4. Tables

Tables are used to organize and present data in rows and columns.

Basic Table Structure

<table> <tr> <th>Header 1</th> <th>Header 2</th> </tr> <tr> <td>Row 1, Cell 1</td> <td>Row 1, Cell 2</td> </tr> <tr> <td>Row 2, Cell 1</td> <td>Row 2, Cell 2</td> </tr> </table>

Semantic Table Elements

Element Description
<caption> Table title or description
<thead> Group of header rows
<tbody> Group of content rows
<tfoot> Group of footer rows
<colgroup> Group of columns for formatting

Semantic Table Example

<table> <caption>Monthly Savings</caption> <thead> <tr> <th>Month</th> <th>Savings</th> </tr> </thead> <tbody> <tr> <td>January</td> <td>$100</td> </tr> <tr> <td>February</td> <td>$80</td> </tr> </tbody> <tfoot> <tr> <td>Sum</td> <td>$180</td> </tr> </tfoot> </table>

Cell Spanning

Cells can span multiple rows or columns:

<table> <tr> <th>Name</th> <th colspan="2">Contact</th> </tr> <tr> <td>John Doe</td> <td>Phone: 555-1212</td> <td>Email: john@example.com</td> </tr> <tr> <td rowspan="2">Jane Smith</td> <td>Phone: 555-3434</td> <td>Email: jane@example.com</td> </tr> <tr> <td>Alt Phone: 555-6789</td> <td>Work: jsmith@work.com</td> </tr> </table>
