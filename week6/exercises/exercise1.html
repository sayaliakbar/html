Exercise 1: Working with Specialized Input Types

Task:

Create a new HTML file called "specialized-inputs.html" in the exercises folder and implement a form that uses various HTML5 specialized input types. Your form should:

  1. Include at least 8 different HTML5 input types
  2. Use appropriate labels for all form controls
  3. Group related inputs using fieldsets and legends
  4. Include submit and reset buttons

Hint:

HTML5 provides specialized input types like email, tel, date, time, number, range, color, url, search, and more. Each offers unique user interface features and built-in validation when supported by the browser.

Expected Result

Your form should showcase different specialized input types, demonstrating how each one provides a unique interface tailored to the type of data being collected.

Check Your Work

After completing the exercise:

  1. Open your HTML file in a web browser
  2. Test each input type to see how it functions
  3. Try different browsers to see how input types are rendered differently
  4. Verify that all inputs have proper labels
  5. Check that related inputs are grouped with fieldsets
  6. Validate your HTML using the W3C Markup Validation Service

Submission Guidelines

When submitting your work, be prepared to:

  1. Explain your choice of input types for different data
  2. Discuss browser compatibility considerations
  3. Share any insights or challenges you encountered
